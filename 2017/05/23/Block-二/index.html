3. 循环引用及解决

注意上面一小段代码blk是局部变量，实际工程中大部分都是全局或者属性。最简单的为例：
typedef void (^blk) (void);
{blk blk; int val} //全局变量 blk和val;

上面这一句代码存在两处循环引用。self和blk 互相引用；val实际上也截获了self，self->val,最终还是self和blk互相引用。通常的解决是使用弱引用__weak。
当然不止两个对象互相持有，复杂的三个或四个对象形成循环引用。

其实在《Objectiv-C高级编程 iOS与OS X多线程和内存管理》中关于Blocks 的实现写的分清楚，（ps C++部分还给老师了）,其中有这样一句实际上当ARC有效时，大多数情形下编译器会恰当地进行判断，自动生成将Block从栈复制到堆上的代码。自动生成通过objc4运行时runtime/objc-arr.mm的_Block_copy函数。
OC代码编译C++代码结构体中存在isa指向内存区域分为：

类 设置对象的存储域
NSConcreteStackBlock 栈
NSConcreteGlobalBlock 数据区域
NSConcreteMallocBlock 堆

通过po调试代码输出block对象也会有诸如<__NSGlobalBlock__: 0x10d7660e0>的字眼。

block 本身在MRC下可能存在以上三个区域中。ARC下可能在数据区域或堆中,大多数在堆中。为什么非要费死劲弄到堆上，因为栈上的Block，如果其所属的变量作用域结束，该Block就被编译器废弃。本来这种事交给程序员（MRC）处理放堆上，苹果的runtime&&ARC替程序员干了。
